Introduction


This user guide describes Shape2Earth Version 1.47 and later.

Welcome to Shape2Earth for MapWindow GIS!  Shape2Earth is the easiest and cheapest way to convert GIS data into Keyhole Markup Language (KML) for viewing in Google Earth.

With Shape2Earth for MapWindow, you can create advanced symbology in MapWindow, which will be used by Shape2Earth to create your KML file.  You can also create information balloons to help convey information to viewers.

Getting to Know Shape2Earth

The following describes the user interface and some of the features available in Shape2Earth:

KML Tab

The KML Tab allows users to select some basic options related the the KML file that will be generated

  1. Layer Name - Use this to set the name that will be seen in the Google Earth "Places" when you load the KML file
  2. Opacity - Use this to set the percentage of opacity for each KML Placemark.
  3. Document : Open - When this is checked, the KML Document will be expanded in the Google Earth "Places" tree.
  4. Doucment Locked - When this is checked, the KML Document will be closed, and will not be able to be opened.
  5. Altitude Mode - Select the value to be used to se the KML <altitudeMode>.  The 3D source options area not available if "Clamped to Ground" is selected
  6. Unit of measure - Sets the units which will be used for height values (Feet or Meters).
  7. By Attribute - When this is selected, you can choose a numeric attribute field from the shapefile, whose values will be used for height.
  8. By Input - When this is selected, the single numeric value entered by the user will be applied to all points.
  9. By Z Value - When this is selected, the Z values from the shapefile point will be used (if the data is present).
  10. Extrude Features - When checked, the 3D feature will be connected to the ground (see <extrude>)
  11. Tessellate Features - When checked, the linear feature will follow the ground terrain (see <tessellate>).
Properties Tab

The Properties Tab provides additional functionality for presenting geospatial features in Google Earth.  In the example below-right, a polygon shapefile is being exported. The right example is a point.

  1. Fill Polygon - If checked, the polygon will be filled.  If unchecked, the polygon will be hollow with only its lines showing.
  2. Create Labels - If checked, Shape2Earth will create a centroid point from the polygon to use as a label.
  3. Labels At Height - Labels At Height- Only visible when "Create Label" is checked.  When checked, the point for the label will be at the same height as the top of the polygons.If unchecked, the point will be on the ground.
  4. Point Options - Use Map Color - When a point shapefile is being converted, the color that MapWindow has assigned to each point will be assigned to that point in KML.
  5. Style - Styles here are specific to points and labels. Styling for lines and polygons are defined in MapWindow GIS.  KML allows you to assign point information to lines and polygons as well. Clicking on the icon in this form will open the Icon Selector.
  6. Style - Label - Select the size of the label text. The default is 1.
  7. Style - Label - Color - Set the color of the label text.
  8. Style - Icon - Select the size of the icon.  the default is .8.
  9. Create Label Rollover - .When this is checked, Shape2Earth will write a StyleMap for these features.  A style map will allow the user to define a seperate style that shows when the point is hovered over in Google Earth.
  10. Style - Rollover - .This defines the style that will be applied to the point feature when the user hovers over it in Google Earth.

Icon Selector for Google Earth or custom point icons

  1. Add Custon Icon - Allows you to add an icon from your hard drive.  The selected icon will show up in the icon window, and will be available later.
  2. Clear Custon Icons - Clears all custom icons that have been loaded.
  3. OK - Changes the current Shape2Earth feature Icon to the Icon selected on this form.
  4. Cancel - Closes thie form without changing the Shape2Earth feature icon.

GIS Data Tab

The GIS Data Tab provides options for embedding Shapefile attribute data in the KML file

  1. Name Field - Use this drop down box to select a GIS attribute field to get values to populate the KML Placemark Name  This is the value used for labels. You will see this value next to Google Earth points, and in the Google Earth .
  2. Snippet Field - Use this drop down box to select a GIS attribute field to get values to populate the KML Snippet.  This value is also used as GeoRSS Link Field if the data is exported to GeoRSS.
  3. Folder Field - If you would like to sort data into Folders, select the attribute field that will be used to to define the Folder values. If there is no value in this field, Folders will not be used.
  4. Order by Feature Name - When checked, KML features will be sorted in alphabetical order using the Name Field value.
  5. Order By Folder - Only available when a Folder Field is selected, this option will sort the Folders in alphabetical order using the Folder Field value.
  6. Select Attributes to Write to KML - Each checked attribute will be written into the KML Description, and will appear in the Google Earth pop-up balloon.

Query Tab

The Query Tab allows users to view Shapefile attribute data, and query for features with specific values

  1. Load Attribute Table - The shapefiles attribute table is not automatically loaded.  Click the "Load Attribute Table" button to load.
  2. "Search For" field - You can search for a specific value in the table by typing in this text field and then clicking "Search".
  3. Show Selected - Click this if you only want see the selected items in the table. You can return to the full view by clickingShow All.
  4. Zoom to Selected - This will zoom you to the selected feature in MapWindow and in Google Earth.

Options Tab

The Options Tab provides options for what spatial features will be exported, and additonal options on how Folders will behave.

  1. Export All - This is the defauls setting, and means that all features will be exported to KML.
  2. Export Selected - When selected, only the features that have been selected in MapWindow using the selection tool will be exported to KML.
  3. Export Current View - When selected, only the features visible in the MapWindow current View will be exported to KML.
  4. Current View Options - Use Only available when "Export Current View" is selected, determines if the features need to be completetely in or only be intersected by the MapWindow View.
  5. Folder Options Open - If Folders are being used to sort data, checking this means the folders will be open (expanded).
  6. Folder Options Locked - If Folders are being used to sort data, checking this means the folders will be locked (they will be closed, and users will not be able to expand them.
  7. GeoRSS Options - Export Description - If exporting to GeoRSS, this will export the HTML description into your GeoRSS file in the Item tag.
  8. GeoRS Options - Export GIS Attributes -  If exporting to GeoRSS, this will export the Field value and types as custom extensions to the GeoRSS file in the Item tag.
  9. Projection Information - For informational purposes, this is the proj4 string used to represent the shapefiles projection information.

Balloon Tab

The Balloon Tab provides options and customizations to the Google Earth data balloon

  1. Name Selected Template - Use this to select the XML template you will use for your balloon.
  2. Organization Name - The name of your organization.
  3. Organization Website - Your organizations website.
  4. Organization Icon - The URL of an icon representing your organization.
  5. Header Picture URL- A URL to a graphic that will act as the balloon header.
